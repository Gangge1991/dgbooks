给GitHub开源项目提交PR的步骤包括：Fork原项目到自己的GitHub账号下，然后克隆到本地，在本地创建一个新分支进行修改，将修改提交到本地仓库并推送到自己的GitHub仓库，最后在GitHub上点击“Create pull request”按钮来提交PR。 
步骤详解
Fork项目
访问你想贡献的开源项目在GitHub上的主页。
点击右上角的“Fork”按钮，在自己的GitHub账号下创建一个该项目的副本。
克隆和准备
将你Fork的项目克隆到本地计算机上，以便进行修改。
为了同步原项目（上游仓库）的最新代码，建议使用git remote add upstream <原项目仓库地址>命令将其添加为上游仓库，并定期使用git fetch upstream和git merge upstream/main命令保持本地同步。
创建一个新的分支用于开发，而不是直接在 main 分支上修改。
进行修改
在新创建的分支上，使用你喜欢的编辑器修改代码。
在完成修改后，使用 git add . 命令暂存所有更改。
使用 git commit -m "你的提交信息" 命令将更改提交到本地仓库，提交信息应清晰描述你做了什么。
推送更改
将你的本地提交推送到你自己在GitHub上的Fork版本，使用 git push origin <你的分支名> 命令。
提交Pull Request
回到你在GitHub上的Fork仓库主页。
通常会看到一个提示，告诉你刚刚推送了一个新分支，并有一个“Compare & pull request”或“New pull request”的按钮，直接点击它。
在创建PR的页面，填写清晰的标题和描述，说明你提交的更改内容。
检查无误后，点击“Create pull request”按钮进行提交。
等待反馈和合并
提交PR后，项目维护者会审查你的代码。
根据他们的反馈进行修改，如果需要，你可以在自己的分支上继续修改并再次推送，PR会自动更新。
一旦PR被批准，维护者会将你的代码合并到原项目中，此时你的PR就完成了
